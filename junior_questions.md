### M1. `__init__.py` — что это такое и для чего он нужен?

<details>
<summary>Подсказка</summary>

**`__init__.py` — это файл, который превращает обычную директорию в Python-пакет.**

- **Исторически**: до Python 3.3 наличие файла `__init__.py` (даже пустого) было **обязательным**
  условием, чтобы `import` считал папку пакетом. Без него Python видел только отдельные модули.

- **В современном Python (3.3+)**:
    - **Regular packages** — пакеты с `__init__.py`. Интерпретатор выполняет этот файл при первом
      импорте пакета (`import package` или `from package import ...`).
    - **Namespace packages** (PEP 420) — пакеты **без** `__init__.py`. Полезны для плагинов, когда
      несколько дистрибутивов хотят "расширять" одно пространство имён.

**Что обычно размещают в `__init__.py` (senior-level практики):**

1. **Инициализацию пакета** — код, который должен выполниться при импорте всего пакета (настройка
   логгинга, регистрация сигналов, импорт side-effects).
2. **Публичный API** — через переменную `__all__` и re-exports:
   ```python
   from .core import Processor
   from .utils import helper
   
   __all__ = ['Processor', 'helper', 'constants']
    ```

</details>

### M2. Что такое пакет в Python? Чем пакет отличается от модуля?

<details>
<summary>Подсказка</summary>

**Пакет (package) — это способ организации нескольких модулей в иерархическую структуру.**

- **Модуль** — это один `.py`-файл (или C-расширение), который можно импортировать.
- **Пакет** — это **директория**, которая сама может быть импортирована и может содержать модули +
  подпакеты.

**Два типа пакетов:**

1. **Regular package** — директория с файлом `__init__.py`.
2. **Namespace package** — директория (или несколько) **без** `__init__.py` (PEP 420). Используется
   для распределённых пакетов (например, `google.cloud.*`).

**Ключевые свойства пакета:**

- Имеет атрибут `__path__`.
- Поддерживает относительные импорты (`from . import submodule`).
- Может содержать подпакеты любой глубины.
- Устанавливается как единое целое через `pyproject.toml` / `setup.py`.

**Когда использовать пакет:**

- Логическая группировка кода.
- Чистый публичный API.
- Избежание конфликтов имён.
- Распространение как библиотеку на PyPI.

</details>

### M3. Что подразумевается под стандартной библиотекой Python?

<details>
<summary>Подсказка</summary>

**Стандартная библиотека (stdlib) — это набор модулей и пакетов, которые поставляются вместе с
интерпретатором Python и доступны «из коробки» без `pip install`.**

Философия Python — **«Batteries Included»**.

**Особенности:**

- Версионируется вместе с Python.
- Часть модулей написана на чистом Python, часть — на C (для производительности).
- Гарантированная совместимость внутри одной мажорной версии.

**Примеры по категориям:**

- **Core**: `os`, `sys`, `math`, `datetime`, `collections`, `dataclasses`
- **Данные**: `json`, `csv`, `pickle`, `struct`
- **Сеть**: `http`, `urllib`, `socket`, `asyncio`, `ssl`
- **Файловая система**: `pathlib`, `shutil`, `tempfile`, `glob`
- **Параллелизм**: `threading`, `multiprocessing`, `concurrent.futures`
- **Тестирование**: `unittest`, `pdb`, `tracemalloc`

**Важный момент:** stdlib не включает популярные библиотеки вроде `requests`, `numpy`, `pandas` —
они устанавливаются отдельно.

</details>

### M4. Зачем нужен OrderedDict если 3.7+ dict is ordered

<details>
<summary>Подсказка</summary>

```python
d1 = {'a': 1, 'b': 2, 'c': 3}
d2 = {'c': 3, 'b': 2, 'a': 1}          # тот же dict, но другой порядок вставки

od1 = collections.OrderedDict([('a',1), ('b',2), ('c',3)])
od2 = collections.OrderedDict([('c',3), ('b',2), ('a',1)])

print(d1 == d2)      # True   ← dict сравнивает ТОЛЬКО содержимое
print(od1 == od2)    # False  ← OrderedDict сравнивает содержимое И ПОРЯДОК
print(d1 == od1)     # True   ← dict и OrderedDict сравниваются "по-старому"
```

**Это критично, когда порядок важен для логики (например, конфиги, кэши, сериализация, тесты).**

```python
od = collections.OrderedDict([('a',1), ('b',2), ('c',3)])

od.move_to_end('b')                    # переместить ключ в конец
od.move_to_end('a', last=False)        # переместить в начало

od.popitem(last=True)   # LIFO — как stack (последний вставленный)
od.popitem(last=False)  # FIFO  — как queue (первый вставленный)
```
**У обычного dict таких методов нет (придётся делать вручную через del + d[key] = ...).**

- **4. Историческая совместимость и legacy-код**
   - Много библиотек и старого кода (до 3.7) явно использовали OrderedDict.
   - Чтобы не ломать всё это, его оставили.
- **5. Небольшие различия в производительности и памяти**
   - dict — чуть быстрее и экономнее по памяти.
   - OrderedDict внутри использует двусвязный список (doubly-linked list) для поддержания порядка, поэтому немного тяжелее.
  
  
 «С Python 3.7 обычный dict гарантированно ordered по спецификации языка. OrderedDict оставили ради явной семантики, дополнительных методов (move_to_end, popitem(last=...)) и строгого сравнения по порядку. В 99 % случаев сейчас достаточно обычного dict.»
</details>
